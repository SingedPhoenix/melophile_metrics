# Portfolio Screenshot Plan

Screenshots should be reviewed before publishing because the app can display personal listening history, playlist names, and account-connected data.

Use these filenames if adding images to this folder:

| Filename | Alt text | Recommended screen |
| --- | --- | --- |
| `main-navigation.png` | Melophile Metrics main navigation with nine bento sections | Home screen showing the primary bento navigation |
| `listening-rankings.png` | Recent listening rankings for tracks, artists, or albums | Pulse or Last rankings view |
| `gem-mines.png` | Gem Mines ranking tiers for music elements | Gem Mines screen with gem-tier pills and ranked list |
| `past-tense-playlists.png` | Release-year playlist family organized by year | Past Tense playlist grid |
| `fresh-discovery.png` | Fresh discovery workflow with seed or harvest playlists | Fresh seed or harvest view |
| `data-integrity.png` | Last.fm data integrity and synchronization panel | Settings screen showing sync/data-integrity controls |
| `listening-clock.png` | Listening clock or time-based visualization | Dashboard visualization screen |

Before adding screenshots, confirm:

1. No private API keys, tokens, email addresses, or account settings are visible.
2. Personal listening data shown in the image is acceptable for public presentation.
3. The image filename matches the README reference if embedded later.
4. The image is cropped cleanly and readable on GitHub.
